---
title: InlineSelect
description: Picker that expands its options in place.
---

A select that reveals its options next to the trigger instead of taking over the screen with a sheet.

## Installation

InlineSelect ships with the library — no separate install.

```tsx
import { InlineSelect } from 'panelui-native';
```

## Usage

```tsx
// Inline: the list expands in layout flow and pushes content below it down.
<InlineSelect value={region} onValueChange={setRegion} placeholder="Select a region">
  <InlineSelect.Item value="us" label="United States" />
  <InlineSelect.Item value="eu" label="Europe" />
</InlineSelect>

// Overlay: the list floats above the page. Nothing else moves.
<InlineSelect overlay value={region} onValueChange={setRegion}>
  <InlineSelect.Item value="us" label="United States" />
  <InlineSelect.Item value="eu" label="Europe" />
</InlineSelect>

// Size the floating list to its longest option instead of the trigger.
<InlineSelect overlay overlayWidth="content" value={region} onValueChange={setRegion}>
  <InlineSelect.Item value="apac" label="Asia Pacific" />
</InlineSelect>
```

## Composition

```tsx
<InlineSelect>
  <InlineSelect.Item value="…" label="…" />
</InlineSelect>
```

- **`InlineSelect.Item`** — One option. `value` identifies it, `label` is what is shown.

## Examples

### Overlay presentation

The list floats above the page, so opening it does not push the content below it down. This is what you want inside a form or a scroll view.

```tsx
const [region, setRegion] = useState<string>();

<InlineSelect
  overlay
  value={region}
  onValueChange={setRegion}
  placeholder="Select a region"
>
  <InlineSelect.Item value="us" label="United States" />
  <InlineSelect.Item value="eu" label="Europe" />
  <InlineSelect.Item value="apac" label="Asia Pacific" />
</InlineSelect>
```

### Inline presentation

The default. The list expands in layout flow and everything below moves down — right inside a settings list, where it reads as the row growing.

```tsx
<InlineSelect value={region} onValueChange={setRegion}>
  <InlineSelect.Item value="us" label="United States" />
  <InlineSelect.Item value="eu" label="Europe" />
</InlineSelect>
```

### Sizing the floating list

```tsx
{/* Match the trigger — the default. */}
<InlineSelect overlay overlayWidth="trigger" {...props} />

{/* Size to the longest option, never narrower than the trigger. */}
<InlineSelect overlay overlayWidth="content" {...props} />

{/* Fixed width, and a wider gap from the trigger. */}
<InlineSelect overlay overlayWidth={280} overlayOffset={12} {...props} />
```

### Reacting to open and close

```tsx
<InlineSelect
  overlay
  value={region}
  onValueChange={setRegion}
  onOpenChange={(open) => setScrollEnabled(!open)}
>
  {/* …items… */}
</InlineSelect>
```

## Variants

### `selected`

- `true`

```tsx
<InlineSelect selected="true">…</InlineSelect>
```

### `presentation`

- `inline` *(default)*
- `overlay`

```tsx
{/* Expands in layout flow — content below moves down. */}
<InlineSelect>…</InlineSelect>

{/* Floats above the page — nothing moves. */}
<InlineSelect overlay>…</InlineSelect>
```

## API Reference

### InlineSelect.Item

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `value` | `string` | — |  |
| `label` | `string` | — |  |

### InlineSelect

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `className` | `string` | — |  |
| `value` | `string` | — |  |
| `onValueChange` | `(value: string) => void` | — |  |
| `placeholder` | `string` | `'Select an option'` |  |
| `disabled` | `boolean` | — |  |
| `overlay` | `boolean` | `false` | Float the list above the page instead of expanding it in layout flow, so opening the select does not push the content below it down. |
| `overlayWidth` | `'trigger' \| 'content' \| number` | `'trigger'` | Width of the floating list. `trigger` matches the trigger, `content` sizes to the longest option, or pass a pixel value. Ignored without `overlay`. |
| `overlayOffset` | `number` | `8` | Gap between the trigger and the floating list. Ignored without `overlay`. |
| `onOpenChange` | `(open: boolean) => void` | — | Called when the list opens or closes. |

Every part also accepts the underlying React Native props (`ViewProps` or `TextProps`) and a `className` for Tailwind utilities.

## Notes

### Choosing a presentation

By default the list expands **in layout flow**, so everything below the select moves down. That reads correctly inside a settings list, where it looks like the row itself growing — and badly anywhere the shift is jarring.

Pass `overlay` to float the list above the page through a portal instead. It is anchored to the trigger, flips above it when there is not enough room below, scrolls when the options do not fit, and closes on a press anywhere outside. **Nothing on the screen moves.** This is what you want inside a form or a scroll view.

### Anchoring

The floating list is positioned from a window-space measurement of the trigger taken when it opens, so it is correct inside scroll views and transformed parents. `overlayWidth` matches the trigger by default; `"content"` sizes to the longest option (never narrower than the trigger) and a number sets a fixed width. `overlayOffset` is the gap between the two.

### Compared to Select

`Select` opens a bottom sheet, which suits long option lists and small screens. `InlineSelect` keeps the options next to what they belong to, which suits short lists inside a form.
