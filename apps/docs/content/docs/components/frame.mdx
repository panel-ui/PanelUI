title: Frame

A widget shell: a titled header row with an action on the right, an inset panel of rows, and a caption footer. The panel's radius is derived from the shell's, so the nesting stays proportional as the theme changes it.

## Installation

Frame ships with the library — no separate install.

```tsx
import { Frame, Button, Switch, Text, Badge } from 'panelui-native';
```

## Usage

```tsx
<Frame>
  <Frame.Header>
    <Frame.Title>Usage Type</Frame.Title>
    <Frame.Action>Amount</Frame.Action>
  </Frame.Header>
  <Frame.Panel>
    <Frame.Row>
      <Text className="flex-1">Total API Requests</Text>
      <Text weight="medium">33.1K</Text>
    </Frame.Row>
    <Frame.Row divided>
      <Text className="flex-1">Input Tokens</Text>
      <Text weight="medium">98.2M</Text>
    </Frame.Row>
  </Frame.Panel>
  <Frame.Footer>Updated 2 minutes ago</Frame.Footer>
</Frame>
```

## Composition

```tsx
<Frame>
  <Frame.Header>
    <Frame.Title>…</Frame.Title>
    <Frame.Action>…</Frame.Action>
  </Frame.Header>
  <Frame.Panel>
    <Frame.Row>…</Frame.Row>
  </Frame.Panel>
  <Frame.Footer>…</Frame.Footer>
</Frame>
```

- **`Frame.Header`** — Row holding the title and the action.
- **`Frame.Title`** — Frame heading.
- **`Frame.Action`** — Trailing slot — a label, a button, a badge. Strings render muted.
- **`Frame.Description`** — Secondary line, inside a column-wrapped header.
- **`Frame.Panel`** — The inset card holding the rows.
- **`Frame.Row`** — A row inside the panel. Pass `divided` on every row after the first.
- **`Frame.Footer`** — Muted caption under the panel. Strings are wrapped for you.

## Examples

### A settings panel

`Frame.Row` with `divided` draws the hairline between rows for you.

```tsx
<Frame>
  <Frame.Header>
    <Frame.Title>Notifications</Frame.Title>
    <Frame.Description>How we reach you.</Frame.Description>
  </Frame.Header>
  <Frame.Panel>
    <Frame.Row divided>
      <Text>Push</Text>
      <Switch value={push} onValueChange={setPush} />
    </Frame.Row>
    <Frame.Row divided>
      <Text>Email</Text>
      <Switch value={email} onValueChange={setEmail} />
    </Frame.Row>
    <Frame.Row>
      <Text>SMS</Text>
      <Switch value={sms} onValueChange={setSms} />
    </Frame.Row>
  </Frame.Panel>
</Frame>
```

### With a header action and a footer

```tsx
<Frame>
  <Frame.Header>
    <Frame.Title>API keys</Frame.Title>
    <Frame.Action>
      <Button size="sm" variant="outline">New key</Button>
    </Frame.Action>
  </Frame.Header>
  <Frame.Panel>
    {keys.map((key) => (
      <Frame.Row key={key.id} divided>
        <Text>{key.name}</Text>
        <Badge variant="outline">{key.lastUsed}</Badge>
      </Frame.Row>
    ))}
  </Frame.Panel>
  <Frame.Footer>
    <Text size="sm" muted>Keys are shown once at creation.</Text>
  </Frame.Footer>
</Frame>
```

## API Reference

### Frame

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `className` | `string` | — |  |

### Frame.Row

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `divided` | `boolean` | — |  |

Every part also accepts the underlying React Native props (`ViewProps` or `TextProps`) and a `className` for Tailwind utilities.

## Notes

React Native has no `:first-child`, so `divided` is set explicitly on each row after the first rather than inferred.
