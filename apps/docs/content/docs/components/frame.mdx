A widget shell: a titled header row with an action on the right, an inset panel of rows, and a caption footer. The panel draws the hairlines between its own rows, and its radius is derived from the shell‚Äôs, so the nesting stays proportional as the theme changes it.

## Installation

Frame ships with the library ‚Äî no separate install.

```tsx
import { Frame, Button, Switch, Text, Badge, Card } from 'panelui-native';
```

Or copy the source into your project, to own and edit it:

```bash
npx panelui-cli@latest add frame
```

## Usage

```tsx
<Frame>
  <Frame.Header>
    <Frame.Title>Usage Type</Frame.Title>
    <Frame.Action>Amount</Frame.Action>
  </Frame.Header>
  <Frame.Panel>
    <Frame.Row>
      <Text className="flex-1">Total API Requests</Text>
      <Text weight="medium">33.1K</Text>
    </Frame.Row>
    <Frame.Row>
      <Text className="flex-1">Input Tokens</Text>
      <Text weight="medium">98.2M</Text>
    </Frame.Row>
  </Frame.Panel>
  <Frame.Footer>Updated 2 minutes ago</Frame.Footer>
</Frame>
```

## Composition

```tsx
<Frame>
  <Frame.Header>
    <Frame.Title>‚Ä¶</Frame.Title>
    <Frame.Action>‚Ä¶</Frame.Action>
  </Frame.Header>
  <Frame.Panel>
    <Frame.Section title="‚Ä¶">
      <Frame.Row>‚Ä¶</Frame.Row>
    </Frame.Section>
  </Frame.Panel>
  <Frame.Footer>‚Ä¶</Frame.Footer>
</Frame>
```

- **`Frame.Header`** ‚Äî Row holding the title and the action.
- **`Frame.Title`** ‚Äî Frame heading.
- **`Frame.Action`** ‚Äî Trailing slot ‚Äî a label, a button, a badge. Strings render muted.
- **`Frame.Description`** ‚Äî Secondary line, inside a column-wrapped header.
- **`Frame.Panel`** ‚Äî The inset card holding the rows. Divides them for you.
- **`Frame.Section`** ‚Äî A labelled cluster of rows, for a panel holding more than one group.
- **`Frame.Row`** ‚Äî A row inside the panel. Give it an `onPress` and it becomes a pressable.
- **`Frame.Footer`** ‚Äî Muted caption under the panel. Strings are wrapped for you.

## Examples

### A settings panel

Nothing has to say where the hairlines go ‚Äî the panel puts one above every row but the first.

```tsx
<Frame>
  <Frame.Header>
    <Frame.Title>Notifications</Frame.Title>
    <Frame.Description>How we reach you.</Frame.Description>
  </Frame.Header>
  <Frame.Panel>
    <Frame.Row>
      <Text className="flex-1">Push</Text>
      <Switch value={push} onValueChange={setPush} />
    </Frame.Row>
    <Frame.Row>
      <Text className="flex-1">Email</Text>
      <Switch value={email} onValueChange={setEmail} />
    </Frame.Row>
    <Frame.Row>
      <Text className="flex-1">SMS</Text>
      <Switch value={sms} onValueChange={setSms} />
    </Frame.Row>
  </Frame.Panel>
</Frame>
```

### With a header action and a footer

```tsx
<Frame>
  <Frame.Header>
    <Frame.Title>API keys</Frame.Title>
    <Frame.Action>
      <Button size="sm" variant="outline">New key</Button>
    </Frame.Action>
  </Frame.Header>
  <Frame.Panel>
    {keys.map((key) => (
      <Frame.Row key={key.id}>
        <Text>{key.name}</Text>
        <Badge variant="outline">{key.lastUsed}</Badge>
      </Frame.Row>
    ))}
  </Frame.Panel>
  <Frame.Footer>
    <Text size="sm" muted>Keys are shown once at creation.</Text>
  </Frame.Footer>
</Frame>
```

### Rows that lead somewhere

A row with an `onPress` is a real pressable ‚Äî press feedback and a button role ‚Äî instead of a View with a handler bolted on. `chevron` says so before you tap it.

```tsx
<Frame.Panel>
  {sections.map((section) => (
    <Frame.Row
      key={section.id}
      chevron
      onPress={() => router.push(section.href)}
    >
      <Text size="sm" className="flex-1">{section.label}</Text>
    </Frame.Row>
  ))}
</Frame.Panel>
```

### Sections inside a panel

For a widget with more than one cluster of rows. Each section labels its group and divides its own rows.

```tsx
<Frame.Panel>
  <Frame.Section title="General">
    <Frame.Row>‚Ä¶</Frame.Row>
    <Frame.Row>‚Ä¶</Frame.Row>
  </Frame.Section>
  <Frame.Section title="Danger zone">
    <Frame.Row chevron onPress={confirmDelete}>
      <Text size="sm" className="flex-1 text-destructive">
        Delete workspace
      </Text>
    </Frame.Row>
  </Frame.Section>
</Frame.Panel>
```

## Variants

### `variant`

- `default` *(default)*
- `plain`

```tsx
{/* The default: a shell with the panel inset inside it. */}
<Frame variant="default">‚Ä¶</Frame>

{/* No shell ‚Äî for a Frame inside a card that already draws a border,
    where the shell‚Äôs own edge just inside it reads as a double line. */}
<Card>
  <Card.Content className="p-4">
    <Frame variant="plain">
      <Frame.Panel>‚Ä¶</Frame.Panel>
    </Frame>
  </Card.Content>
</Card>
```

Every part also accepts the underlying React Native props (`ViewProps` or `TextProps`) and a `className` for Tailwind utilities.

## Notes

React Native has no `:first-child`, so the hairline between rows cannot be a CSS rule. `Frame.Panel` places them instead ‚Äî a line above every child but the first. Passing `divided` on a row overrides that either way, and `dividers={false}` on the panel hands the whole job back to you.

`Frame.Section` divides its own rows the same way, so the two nest without either needing to know about the other.
