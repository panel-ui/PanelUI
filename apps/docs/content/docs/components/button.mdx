The primary action component. Icons in `startContent` and `endContent` inherit a colour that reads against the button's background, so they follow the theme without a hardcoded hex.

## Installation

Button ships with the library — no separate install.

```tsx
import { Button, SearchIcon, ChevronRightIcon, GoogleIcon, AppleIcon } from 'panelui-native';
```

Or copy the source into your project, to own and edit it:

```bash
npx panelui-cli@latest add button
```

## Usage

```tsx
<Button onPress={save}>Save changes</Button>

<Button variant="outline" startContent={<SearchIcon size={16} />}>
  Search
</Button>

<Button variant="social" fullWidth startContent={<GoogleIcon size={18} />}>
  Continue with Google
</Button>

<Button loading={saving} fullWidth>
  {saving ? 'Saving…' : 'Save'}
</Button>
```

## Examples

### With icons

Icons in `startContent` and `endContent` inherit a colour that reads against the button’s background, so they follow the theme without a hardcoded hex.

```tsx
<Button startContent={<SearchIcon size={16} />}>Search</Button>

<Button variant="outline" endContent={<ChevronRightIcon size={16} />}>
  Continue
</Button>

{/* Icon-only buttons get size="icon" and an accessibility label. */}
<Button size="icon" variant="ghost" accessibilityLabel="Search">
  <SearchIcon size={18} />
</Button>
```

### Loading

`loading` swaps the start content for a variant-matched spinner and blocks presses, so you do not need a separate `disabled`.

```tsx
const [saving, setSaving] = useState(false);

async function save() {
  setSaving(true);
  try {
    await api.save(draft);
  } finally {
    setSaving(false);
  }
}

<Button loading={saving} fullWidth onPress={save}>
  {saving ? 'Saving…' : 'Save changes'}
</Button>
```

### A sign-in stack

The `social` variant is a neutral surface sized for a full-width column of providers.

```tsx
<View className="gap-2">
  <Button variant="social" fullWidth startContent={<GoogleIcon size={18} />}>
    Continue with Google
  </Button>
  <Button variant="social" fullWidth startContent={<AppleIcon size={18} />}>
    Continue with Apple
  </Button>
</View>
```

### Destructive confirmation

```tsx
<View className="flex-row gap-2">
  <Button variant="ghost" onPress={cancel}>Cancel</Button>
  <Button variant="destructive" onPress={deleteAccount}>
    Delete account
  </Button>
</View>
```

## Variants

### `variant`

- `primary` *(default)*
- `secondary`
- `outline`
- `ghost`
- `destructive`
- `social`

```tsx
<Button>Primary</Button>
<Button variant="secondary">Secondary</Button>
<Button variant="outline">Outline</Button>
<Button variant="ghost">Ghost</Button>
<Button variant="destructive">Delete</Button>
<Button variant="social" startContent={<GoogleIcon size={18} />}>
  Continue with Google
</Button>
```

### `size`

- `sm`
- `md` *(default)*
- `lg`
- `icon`

```tsx
<Button size="sm">Small</Button>
<Button size="md">Medium</Button>
<Button size="lg">Large</Button>
<Button size="icon" accessibilityLabel="Search">
  <SearchIcon size={18} />
</Button>
```

### `fullWidth`

- `true`

```tsx
<Button fullWidth>Spans its container</Button>
```

## API Reference

### Button

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `disabled` | `boolean` | — |  |
| `loading` | `boolean` | `false` | Show a spinner and block presses while an action is in flight. |
| `startContent` | `ReactNode` | — | Content rendered before the label (replaced by the spinner while loading). |
| `endContent` | `ReactNode` | — | Content rendered after the label. |
| `labelClassName` | `string` | — | Extra classes for the label when children is a string. |
| `native` | `boolean` | — | Render the platform's own button instead of this one. Requires the optional `@expo/ui` package; without it this prop does nothing. **Theme tokens do not apply** — the platform draws the button, so `className`, `fullWidth`, `startContent`, `endContent` and `loading` are all ignored. `variant` maps onto the nearest platform style: `primary`/`destructive` → filled, `outline` → outlined, everything else → text; `size` sets the height. A native button **sizes itself to its label**, the way a platform button is supposed to. It does not stretch to fill its container, and `fullWidth` has no effect on it. |

Every part also accepts the underlying React Native props (`ViewProps` or `TextProps`) and a `className` for Tailwind utilities.

## Notes

`loading` swaps the start content for a variant-matched spinner and blocks presses. The `social` variant is a neutral surface sized for a full-width sign-in stack; pair it with `GoogleIcon`, `FacebookIcon` or `AppleIcon`.

### Native rendering

Pass `native` to render the platform's own button instead — SwiftUI on iOS, Jetpack Compose on Android. It needs the optional `@expo/ui` package and is a silent no-op without it.

**Theme tokens do not apply in native mode**: the platform draws the control with its own colours and metrics, so `className` and most styling props are ignored. `variant` maps onto the nearest platform style; `size`, `fullWidth`, `loading`, `startContent` and `endContent` are dropped.

See [Native rendering](/docs/native) for the full prop-by-prop breakdown.
